# LogiTrack バックアップ手順書

## 概要

このドキュメントでは、LogiTrackシステムのバックアップ方法について説明します。
複数人で作業する場合は、定期的なバックアップとGitでのコード管理が重要です。

---

## バックアップの種類

| 種類 | 対象 | 頻度 | 方法 |
|------|------|------|------|
| コードバックアップ | app.js, index.html等 | 変更時 | Git + PowerShellスクリプト |
| データバックアップ | ロット情報、マスタ等 | 毎日 | JSONエクスポート |
| Firestoreバックアップ | サーバーデータ | 週1回 | Firestoreエクスポート |

---

## 1. コードのバックアップ（Git）

### 変更をコミットする

```powershell
cd D:\Antigravity_Source

# 変更を確認
git status

# すべての変更をステージング
git add .

# コミット（メッセージは変更内容を記載）
git commit -m "機能追加：〇〇を実装"
```

### 変更履歴を確認する

```powershell
git log --oneline -10
```

### 過去のバージョンに戻す（緊急時）

```powershell
# 直前のコミットに戻す（変更は保持）
git reset --soft HEAD~1

# 特定のコミットに戻す（変更は破棄）
git reset --hard コミットID
```

---

## 2. ZIPバックアップ（ローカル/USB）

### 手動でバックアップを作成

PowerShellを開いて以下を実行：

```powershell
cd D:\Antigravity_Source
.\backup.ps1
```

### 自動で行われる処理

1. `D:\Antigravity_Backups` にZIPファイルを作成
2. USBドライブが接続されていれば自動コピー
3. 30日以上前のバックアップを自動削除

---

## 3. データのエクスポート（ブラウザから）

### ローカルデータをエクスポート

1. ブラウザでLogiTrackを開く（http://localhost:3000）
2. F12キーを押してコンソールを開く
3. 以下を入力してEnter：

```javascript
window.exportAllData()
```

4. JSONファイルがダウンロードされます

### Firestoreデータをエクスポート

```javascript
window.exportFromFirestore()
```

### データをインポート（復元）

```javascript
window.importAllData()
```

ファイル選択ダイアログが開くので、エクスポートしたJSONファイルを選択します。

---

## 4. 推奨バックアップスケジュール

### 毎日（作業終了時）

1. Gitでコミット
2. データエクスポート（window.exportAllData()）

### 週1回

1. PowerShellスクリプトでZIPバックアップ
2. USBにコピー
3. Firestoreエクスポート（window.exportFromFirestore()）

### 重要な変更後

1. 必ずGitでコミット
2. ZIPバックアップを作成

---

## 5. 復元手順

### コードを復元する場合

```powershell
# ZIPファイルを解凍
Expand-Archive -Path "D:\Antigravity_Backups\LogiTrack_Backup_日時.zip" -DestinationPath "D:\Antigravity_Source_Restored"
```

### データを復元する場合

1. ブラウザでLogiTrackを開く
2. コンソールで `window.importAllData()` を実行
3. エクスポートしたJSONファイルを選択

---

## 6. トラブルシューティング

### 「バックアップスクリプトが実行できない」

PowerShellの実行ポリシーを変更：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 「Gitコマンドが見つからない」

Git for Windowsをインストールしてください：
https://git-scm.com/download/win

### 「データが古い」

1. Firestoreからエクスポート（最新のサーバーデータ）
2. ローカルにインポート

---

## 7. 複数人で作業する場合の注意

1. **作業開始前**にデータをエクスポートして最新版を確認
2. **作業終了時**に必ずGitでコミット
3. **同時編集は避ける**（同じファイルを複数人で編集しない）
4. **変更内容を共有**（チャットやメールで連絡）
